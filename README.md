# ReaSlideNote

FL-style slide-note pitch control for REAPER.

ReaSlideNote is a small REAPER script set that lets you create pitch slides using dedicated slide marker notes, instead of drawing pitch bend by hand.

It is designed for a workflow similar to FL Studio slide notes:

- slide start is defined by note position
- target pitch is defined by note pitch
- slide time is defined by note length

## Included files

ReaSlideNote includes:

- `ReaSlideNote_Encoder.lua`
- `ReaSlideNote_Bake.lua`
- `AddReaSlideNote.lua`
- `ReaSlideNote.jsfx`

## What it does

ReaSlideNote lets you control pitch movement with MIDI notes.

You place:

- **regular notes** for normal playback
- **slide marker notes** for pitch-slide instructions

A slide marker note defines:

- **when** the slide begins
- **which pitch** the note slides to
- **how long** the slide lasts

This makes it possible to build pitch movement directly in the piano roll, without manually editing pitch bend curves.

## Basic workflow

1. Insert normal notes as usual.
2. Insert slide marker notes on **MIDI channel 16**.
3. The slide marker note pitch defines the target pitch.
4. The slide marker note length defines the slide duration.
5. Keep `ReaSlideNote_Encoder.lua` running.
6. Insert `ReaSlideNote.jsfx` directly before your synth.
7. Match your synth's **pitch bend range** to the JSFX **Bend Range** setting.

## Included scripts

### ReaSlideNote_Encoder.lua
Generates the helper data used by ReaSlideNote.

This script should be kept running while editing.  
For convenience, it is recommended to add it to REAPER startup actions.

### ReaSlideNote_Bake.lua
Bakes the ReaSlideNote result into ordinary pitch bend data on a new track.

This is useful when you want to convert the note-based workflow into standard MIDI pitch bend.

### AddReaSlideNote.lua
Adds `ReaSlideNote.jsfx` to the beginning of the selected track's FX chain.

### ReaSlideNote.jsfx
Reads the helper data generated by the encoder and produces pitch bend during playback.

## Features

- FL-style slide-note workflow inside REAPER
- note-based pitch slide control
- slide timing defined by note length
- target pitch defined by note pitch
- bake-to-pitch-bend workflow
- release-tail slide support
- integer semitone take-pitch support
- same-bend chord movement is possible

## Bake workflow

ReaSlideNote also includes a bake script.

This lets you keep the flexible note-based workflow during editing, then convert the result into ordinary pitch bend data later.

The bake script:

- creates a new track
- copies MIDI note data
- writes baked pitch bend
- leaves the original track untouched

## Installation

### ReaPack
Install the package from ReaPack.

### Manual install
Place the files in your REAPER resource folders:

- Lua scripts in your Scripts folder
- JSFX in your Effects folder

Then:

- add the Lua scripts to the Action List
- run `ReaSlideNote_Encoder.lua`
- insert `ReaSlideNote.jsfx` before your synth

## Recommended setup

For day-to-day use, this setup is recommended:

- keep `ReaSlideNote_Encoder.lua` running
- register the encoder in REAPER startup actions
- use `AddReaSlideNote.lua` to insert the JSFX quickly
- keep your synth bend range matched to ReaSlideNote

## Known limitations

- ReaSlideNote uses **channel-wide pitch bend**, not per-note pitch control.
- It is not intended for complex independent polyphonic pitch movement.
- Slide marker notes are fixed to **MIDI channel 16**.
- Edits are reflected when playback starts again, not as fully live real-time updates.
- Hand-drawn pitch bend cannot be used together with ReaSlideNote.
- Velocity slide is not supported.
- Release slides only work while the synth is still producing audible release.
- When the next base note starts, the previous slide is terminated.

## Best use cases

ReaSlideNote works especially well for:

- mono leads
- synth basses
- glide-heavy phrases
- FL-style note-authored pitch slides
- chords that should move together with the same bend amount

## Notes

ReaSlideNote is best thought of as a note-authored pitch-slide system, rather than a traditional synth legato feature.

Instead of relying on overlap-triggered glide behavior, it lets you place explicit slide notes in the piano roll and define the movement directly.
