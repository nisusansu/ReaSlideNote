# ReaSlideNote

Create FL-style note-authored pitch slides directly in REAPER.

ReaSlideNote is a small REAPER script set that lets you create pitch slides using dedicated slide marker notes instead of drawing pitch bend by hand.

It allows you to define pitch movement directly in the MIDI editor, while the included scripts and JSFX handle the conversion into pitch bend during playback.

## Install with ReaPack

You can install ReaSlideNote via ReaPack by importing this repository:

`https://raw.githubusercontent.com/nisusansu/ReaSlideNote/main/index.xml`

In REAPER, open:

`Extensions > ReaPack > Import repositories...`

Paste the URL above, import the repository, then search for `ReaSlideNote` in ReaPack and install it.

## Overview

ReaSlideNote uses two kinds of notes:

- **regular notes** for normal playback
- **slide marker notes** for slide control

A slide marker note determines:

- **when** the slide starts
- **which pitch** the note slides to
- **how long** the slide lasts

This makes it possible to build pitch movement directly in the piano roll, in a way that is closer to FL-style slide notes than to ordinary synth legato.

## Basic workflow

1. Insert your normal notes as usual.
2. Insert slide marker notes on **MIDI channel 16**.
3. The pitch of each slide marker note defines the target pitch.
4. The length of each slide marker note defines the slide duration.
5. Keep `ReaSlideNote_Encoder.lua` running.
6. Insert `ReaSlideNote.jsfx` directly before your synth.
7. Set your synth’s **pitch bend range** to match the JSFX **Bend Range** setting.

## Included tools

### ReaSlideNote_Encoder.lua
Generates the helper data used by ReaSlideNote.

### ReaSlideNote_Bake.lua
Bakes the ReaSlideNote result into standard pitch bend data on a new track.

### AddReaSlideNote.lua
Adds `ReaSlideNote.jsfx` to the beginning of the selected track’s FX chain.

### ReaSlideNote.jsfx
Reads the helper data generated by the encoder and produces pitch bend during playback.

## Features

- FL-style slide-note workflow inside REAPER
- note-based pitch slide control
- slide timing defined by note length
- target pitch defined by note pitch
- bake-to-pitch-bend workflow
- integer-semitone take pitch support

## Manual installation

If you do not use ReaPack, you can install the files manually:

- place the Lua scripts in your REAPER **Scripts** folder
- place `ReaSlideNote.jsfx` in your REAPER **Effects** folder

Then:

- add the Lua scripts to the Action List
- run `ReaSlideNote_Encoder.lua`
- insert `ReaSlideNote.jsfx` before your synth

## Recommended setup

For convenience, it is recommended that you:

- add the encoder to REAPER’s startup actions
- use `AddReaSlideNote.lua` to insert the JSFX quickly

## Known limitations

- ReaSlideNote uses **channel-wide pitch bend**, not per-note pitch control.
- It is not intended for complex independent polyphonic pitch movement.
- Slide marker notes are fixed to **MIDI channel 16**.
- Edits are applied when playback starts again rather than updating fully in real time.
- Your synth’s pitch bend range must match the JSFX **Bend Range** setting.
- Hand-drawn pitch bend cannot be used together with ReaSlideNote.
- Velocity slide is not supported.

## Best use cases

ReaSlideNote works especially well for:

- mono leads
- synth basses
- glide-heavy phrases
- FL-style note-authored pitch slides
- chords that move together with the same bend amount

## Notes

ReaSlideNote is best understood as a **note-authored pitch-slide system** rather than a traditional synth legato feature.

In other words, regular legato or portamento is usually a synth behavior, while ReaSlideNote makes slide behavior editable as note data in the MIDI editor.
